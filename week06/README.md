```

@NotBlank(message = "이름은 필수 입력 항목입니다.")
@Size(min = 2, max = 20, message = "이름은 2~20자 사이로 입력해주세요.")

        RedirectAttributes redirectAttributes) {

    if (result.hasErrors()) {
        return "student/addForm"; // 검증 실패 → 폼 재렌더링
    }
    // ...
}
```

### Lab 05 (25분) — XSS 방어 4가지 방법

**목표:** 이스케이프 계층별 방어 방법 비교 체험, allowlist 방식(OWASP) vs 전체 차단(HtmlUtils) 차이 이해

**추가 파일:**
- `build.gradle` — OWASP HTML Sanitizer 의존성 추가
- `templates/student/xssTest.html` — 4가지 방법 비교 박스
- `templates/student/xssFilterTest.html` — 서블릿 필터 레벨 방어 데모
- `StudentController.java` — `xssTest()`에 `HtmlUtils.htmlEscape()` + OWASP sanitize 추가, `filterTest()` 신규
- `config/XssEscapeFilterConfig.java` — XSS Servlet Filter 동작 원리 직접 구현 (config/ 폴더 생성)

**이스케이프 계층별 방어 방법:**

| 방법 | 이스케이프 위치 | 안전 여부 | 특징 |
|------|----------------|-----------|------|
| `th:utext` | 없음 | 위험 | HTML 그대로 렌더링 |
| `th:text` | View 레이어 (Thymeleaf) | 안전 | 가장 간단, Spring MVC 기본 권장 |
| `HtmlUtils.htmlEscape()` | Controller 레이어 (Java) | 안전 | 모든 HTML 태그 차단 (서식도 소실) |
| `OWASP HTML Sanitizer` | Controller 레이어 (Java) | 안전 | allowlist 방식: `<b>`, `<i>` 등 허용, `<script>` 제거 |
| `XssEscapeFilter` | Servlet Filter 레이어 | 안전 | Controller 코드 변경 없이 전역 적용 |

**HtmlUtils vs OWASP 핵심 차이:**
```
입력: <b>굵은글씨</b><script>alert('XSS')</script>

HtmlUtils → &lt;b&gt;굵은글씨&lt;/b&gt;&lt;script&gt;...  (모든 HTML 이스케이프, 서식 소실)
OWASP     → <b>굵은글씨</b>                              (script만 제거, b 태그 보존 → 굵게 표시)
```

**실습 A — xss-test (4가지 방법 비교):**
1. [http://localhost:8080/students/xss-test](http://localhost:8080/students/xss-test) 접속
2. `<b>굵은글씨</b><script>alert('XSS')</script>` 입력 후 제출
3. 결과:
   - `th:utext` 박스: 브라우저 alert 팝업 발생 + 굵은글씨 렌더링 ⚠️
   - `th:text` 박스: `<script>` 문자열로 표시 ✅
   - `HtmlUtils` 박스: `<b>`까지 모두 이스케이프 → 서식 없이 텍스트만 ✅
   - `OWASP` 박스: `<b>굵은글씨</b>` 보존, `<script>` 제거 ✅

**실습 B — filter-test (서블릿 필터 레벨):**
1. [http://localhost:8080/students/filter-test](http://localhost:8080/students/filter-test) 접속
2. `<script>alert('XSS')</script>` 입력 후 제출
3. `StudentController.filterTest()` 메서드에 `HtmlUtils.htmlEscape()` 호출이 없는데도
   `th:utext` 출력 시 스크립트가 실행되지 않음 → `XssEscapeFilter`가 Controller 진입 전에 sanitize

**XSS Servlet Filter:**
- `HttpServletRequestWrapper` Decorator 패턴으로 요청 파라미터를 Controller 진입 전에 자동 sanitize
- XML 룰 파일로 URL/파라미터별 예외 설정 가능
- 본 실습: `XssEscapeFilterConfig.java` — Jakarta EE(Spring Boot 3.x) 호환 직접 구현

- `templates/student/editForm.html` ← 교체 (header/footer fragment 적용)
- `templates/student/xssTest.html` ← 교체 (header/footer fragment 적용)

**fragment 정의 (layout.html):**
```html
<!-- th:fragment="header": "header" 이름의 Fragment로 정의 -->
<header th:fragment="header">
    <nav class="navbar">
        <a th:href="@{/students}">학생 관리</a>
        <a th:href="@{/students/xss-test}">XSS 테스트</a>
    </nav>
</header>
```

**fragment 사용 (각 템플릿):**
```html
<!-- th:replace: 현재 태그를 fragment로 완전 교체 -->
<header th:replace="~{fragments/layout :: header}"></header>
<footer th:replace="~{fragments/layout :: footer}"></footer>

<!-- th:insert: 현재 태그 안에 fragment 삽입 (감싸는 태그 유지) -->
<div th:insert="~{fragments/layout :: header}"></div>
```

## IntelliJ Community Edition에서 week06.http 실행하기

HTTP Client 기능은 **IntelliJ IDEA Ultimate(유료)** 전용입니다.
Community Edition(무료) 사용자는 아래 방법으로 동일하게 실행할 수 있습니다.

> `week06.http` 파일에는 `> {% client.test() / client.assert() %}` 스크립트 블록이 사용됩니다.
> VS Code REST Client(Huachao Mao)는 이 블록을 지원하지 않으니 아래 방법을 사용하세요.

### 방법 1 (추천) — JetBrains ijhttp CLI (공식 바이너리)

JetBrains가 공식 배포하는 독립 CLI — Node.js 불필요, `client.test()` 완전 지원

**설치**

macOS (Homebrew):
```bash
brew install ijhttp
```

Windows / Linux: [https://www.jetbrains.com/ijhttp/download/](https://www.jetbrains.com/ijhttp/download/) 에서 ZIP 다운로드 후 압축 해제

> ZIP 설치 시 JDK 17+ 필요. `java --version`으로 확인

**실행**
```bash
# 파일 내 모든 요청 실행
ijhttp week06/week06.http

# 상세 로그 출력
ijhttp week06/week06.http -L VERBOSE
```

### 방법 2 — httpYac CLI

IntelliJ `.http` 형식을 `client.test()` / `client.assert()` 포함하여 **완전히 지원**합니다.

**설치** (Node.js 18+ 필요)
```bash
npm install -g httpyac
```

**실행**
```bash
# 파일 내 모든 요청 실행
httpyac send week06/week06.http --all

# 특정 요청만 실행
httpyac send week06/week06.http --name "Lab01-1"
```

### 방법 3 — VS Code + httpYac 확장

GUI 환경에서 IntelliJ와 유사하게 사용할 수 있습니다.

**설치**
1. [VS Code](https://code.visualstudio.com/) 설치
2. 확장 마켓플레이스에서 **`httpYac`** (작성자: anweber) 검색 후 설치

**실행**
- `.http` 파일을 열면 각 요청 위에 `Send` 버튼이 나타남
- 오른쪽 패널에 응답 결과와 `client.test()` 통과 여부가 표시됨

### 방법 비교

| 방법 | 비용 | `client.test()` 지원 | Node.js 필요 | GUI |
|------|------|----------------------|--------------|-----|
| IntelliJ Ultimate | 유료 | ✅ | ❌ | ✅ |
| **ijhttp CLI** | 무료 | ✅ | ❌ | ❌ |
| **httpYac CLI** | 무료 | ✅ | ✅ | ❌ |
| **VS Code + httpYac** | 무료 | ✅ | ✅ | ✅ |
| VS Code + REST Client (Huachao Mao) | 무료 | ❌ | ❌ | ✅ |
